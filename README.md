# FC Transaction Management System

A financial transaction management system built with **Node.js**, **Express**, and **React**. It provides a REST API for data operations and a React frontend for displaying, adding, editing, and deleting transactions. All data is persisted in a CSV file.

---

## Tech Stack

| Layer | Technology |
|---|---|
| **Backend** | Node.js, Express.js |
| **Frontend** | React, Vite|
| **Storage** | CSV file |
| **Validation** | Shared package (`@transaction/shared`) |
| **Architecture** | Monorepo via npm workspaces |

---

## Prerequisites

- [Node.js](https://nodejs.org) v18.0.0 or higher
- npm v7.0.0 or higher _(comes with Node.js)_
- [Git](https://git-scm.com)

Verify your versions:
```bash
node --version   # v18.x.x or higher
npm --version    # 7.x.x or higher
```

## Installation

```bash
# 1. Clone the repository
git clone https://github.com/axvolkzki/FC-transaction-management-system.git

# 2. Navigate into the project
cd FC-transaction-management-system

# 3. Install all dependencies (backend + frontend + shared)
npm install
```

> **Note:** A single `npm install` from the root installs everything — no need to run it separately inside `backend/` or `frontend/`.

## Database Migration
 
Before running the application for the first time, run the migration script to ensure all existing records in `transactions.csv` have a unique UUID (v4) assigned:
 
```bash
cd backend
node migrate.js
```
 
Expected output:
```
Migration completed. Migrated total number of transactions with unique IDs.
```
 
> **Why this is needed:** Existing records in the CSV may not have an `id` field. The migration script automatically generates a unique UUID v4 for any record missing one, while safely skipping records that already have an ID. New transactions created after this step will have UUIDs auto-generated by the API.
 
## Running the Application

### Both servers at once (recommended)
```bash
npm run dev
```

### Separately
```bash
npm run dev:backend    # API → http://localhost:3001
npm run dev:frontend   # App → http://localhost:3000
```

| Service | URL |
|---|---|
| Frontend | http://localhost:3000 |
| Backend API | http://localhost:3001 |
| Health Check | http://localhost:3001/health |

> **Important:** Always run the backend before or alongside the frontend. The React app fetches data from the API — without it, pages will appear empty.

## API Endpoints

Base URL: `http://localhost:3001`

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Check if API is running |
| `GET` | `/transactions` | Get all transactions (supports filters) |
| `GET` | `/transactions/:id` | Get a single transaction |
| `POST` | `/transactions` | Create a new transaction |

### Query Parameters for `GET /transactions`

| Parameter | Example | Description |
|---|---|---|
| `status` | `?status=Pending` | Filter by status |
| `search` | `?search=john` | Search by name |
| `from` | `?from=2026-01-01` | Filter from date |
| `to` | `?to=2026-03-31` | Filter to date |

### Example Request

```bash
curl -X POST http://localhost:3001/transactions
  -H "Content-Type: application/json"
  -d '{
    "transaction_date": "2026-03-15",
    "account_number": "5551-0000-1234",
    "account_holder_name": "John Doe",
    "amount": 500.00,
    "status": "Pending"
  }'
```

## Validation Rules

All validation is enforced on both frontend and backend using the shared `@transaction/shared` package.

| Field | Rule |
|---|---|
| `transaction_date` | Required |
| `account_number` | Required, format `XXXX-XXXX-XXXX` |
| `account_holder_name` | Required |
| `amount` | Required, must be a positive number |
| `status` | Required, must be `Pending`, `Settled`, or `Failed` |
